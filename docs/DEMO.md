# Fortis browser demo

`index.html` runs the Fortis engine **client-side** via [Pyodide](https://pyodide.org)
(CPython in WebAssembly). No server: the page fetches `fortis_bundle.zip`, unpacks
the engine + inventories into Pyodide's in-browser filesystem, and derives words
locally. Hosted as static files on GitHub Pages.

## Deploy (one-time)

1. Commit `docs/` to `main` and push.
2. Repo **Settings → Pages → Source: Deploy from a branch**, branch `main`, folder **`/docs`**.
3. Live at `https://krzysztofrbogusz.github.io/Fortis/` within a minute or two.

## Update the engine/lexicon in the demo

The demo runs from `fortis_bundle.zip`, a snapshot of `src/fortis/` + `inventories/`.
Whenever either changes, rebuild and push:

```
python docs/build_bundle.py
git add docs/fortis_bundle.zip && git commit -m "Rebuild web bundle" && git push
```

## Files

| file | role |
|---|---|
| `index.html` | the page — UI + Pyodide bootstrap, self-contained |
| `build_bundle.py` | bundles `src/fortis/` + `inventories/` → `fortis_bundle.zip` |
| `fortis_bundle.zip` | the snapshot the page fetches (regenerated by the script) |
| `.nojekyll` | serve files verbatim (skip Jekyll's Markdown processing) |

## Note on a fuller site (Zola etc.)

This is the bare interactive demo. If you later want a polished site around it —
landing page, the rendered `user_guide.md` / `features.md`, examples — a static-site
generator like **Zola** builds that, deployed to Pages via a GitHub Action. The demo
page drops in unchanged as a static asset; nothing here needs to be redone.
